Portfolio Tracker — User Guide

A self-hosted app to track your net worth across stocks, crypto, cash, stablecoins, income, and expenses. Replaces the legacy Excel spreadsheet.

Key Concepts

Accounts

An account is where you hold assets — a brokerage, a bank, a crypto exchange, or a physical asset (e.g. car, house). Each account has a type that classifies it.

Assets

An asset is what you own — a stock (VOO), a crypto token (BTC), a fiat currency (USD), or a stablecoin (USDC). Each asset has a type, optional risk category, and optional target allocation %.

Snapshots

A snapshot is a point-in-time record: "On this date, this account held this quantity of this asset, worth this much USD." Snapshots are the core data — every chart reads from them.

Snapshots are not created automatically unless the pricefeed is running. You can create them manually by clicking "snapshot" in the navigation bar.

Positions

A position is your current holding — what you own right now. Unlike snapshots (historical time-series), positions show the latest state. They include average cost for P&L calculation.

Allocation Targets

Targets define what percentage of your total net worth should be in each category (stocks 45%, stables 43%, crypto 5%, etc.). The app computes how much to buy/sell to reach your targets.

First Steps

  1. Review your data on the Dashboard — check the net worth total makes sense
  2. Go to Data → Assets and deactivate any tokens you no longer own
  3. Go to Positions, unlock editing, and verify your stock holdings (shares + avg cost)
  4. Go to Targets and set your desired allocation percentages
  5. Add your latest Income and Expenses

Action Center (Home)

/ — answers "what should I do this week?" not just "what do I own?".

  • Hero block — Net Worth as the big number, with the 30-day delta below it and a tiny sparkline of the last ~30 days. Three secondary stats: Drift (how far off targets, in pp), Deployed (% of NW working in stocks/crypto/stables), and Mode (active market regime).
  • Action card stack — top 5 ranked actions (rebalance, concentration trim, stale-snapshot warning). Each has a priority rail color (alert-deep red / amber / blue), a headline, the reasoning, and the dollar amount + buy/sell direction. Ordered by an urgency score 0–100.
  • Sentiment strip — compact one-line view of stocks (VOO) + crypto (BTC) market signals plus the active mode. Click "edit targets" to open /targets.
  • Below the fold — Net Worth by Category combo chart and the three doughnuts (Asset Distribution / Stocks / Crypto). They moved here so the homepage's job is decisions, not display.
All Action Center content is fetched in one request from /api/insights/summary. Action thresholds: drift < 2pp or < $1,000 is suppressed; minimum trade size is max($500, 0.5% of NW); concentration warning fires above 25% in a single asset.

Wealth (Holdings)

/wealth — every position you hold across stocks, crypto, stables, cash, and physical assets in a single sortable, filterable table.

  • Filters — all / stocks / crypto / stables / cash / car. Clicking the type pill on the left filters in place.
  • Search — type to filter by symbol, name, or account name. Press / from anywhere on the page to focus the search.
  • Sort — click any column header. Default sort is by value descending.
  • Drift bar — each row shows its parent category's drift from active target. Bar fills toward the right when over-target, left when under. Color stays neutral until ±10pp (warn) or ±20pp (critical).
  • Sparkline — last ~30 daily price closes per asset, drawn in green/red/neutral by direction. Shows "—" if there isn't enough price history yet.

Stocks

/stocks — Stock portfolio and dividend analysis.

  • Dividend charts — monthly, yearly (with growth %), quarterly, year-over-year comparison
  • Portfolio charts — value over time by broker, holdings doughnut, holding value bar, yearly growth rate (green/red)
  • Positions table — your current stock holdings. Edit from /positions.

Crypto

/crypto — Crypto holdings with risk analysis.

  • Value by token — stacked area chart showing each token's value over time
  • Risk breakdown — doughnut showing Safe / Medium / High allocation with percentages
  • Holdings table — symbol, amount, price, value, risk level, target %. Editable (click lock icon).
  • Actual vs Target — grouped bar comparing your real allocation % vs target % for tokens with targets (BTC, SOL)

Allocation Targets

/targets — Set desired portfolio percentages per category, per market mode.

  • Market Signals — sentiment cards per sector (50/200-day MA based) with overall mode
  • Chart — grouped bars comparing your actual % vs the active target %
  • Table — category × Bull / Crab / Bear columns. The active column is highlighted.
Target percentages are decimals: 0.45 = 45%. The adjustment calculation is available at /api/allocation/adjustments. See Market Sentiment for how the active mode is chosen.

Market Sentiment

Sentiment is calculated from real price data — not from your portfolio's recent value changes. It uses two standard institutional indicators: the 50-day and 200-day Simple Moving Average (SMA).

How it works

For each sector with a tradeable representative asset (stocks → VOO, crypto → BTC), the system pulls daily prices from price_history and computes:

  • 50-day MA — average closing price over the last 50 days (short-term trend)
  • 200-day MA — average closing price over the last 200 days (long-term trend)
  • Current price — most recent close

Classification

SignalConditionMeaning
BULL ↑price > 200MA and 50MA > 200MAUptrend confirmed (Golden Cross zone). Favor growth assets.
BEAR ↓price < 200MA and 50MA < 200MADowntrend (Death Cross zone). Favor safety (stables, cash).
CRAB →mixed signalsNo clear direction. Use balanced default allocation.

Data requirements

Real signals require the pricefeed to have been collecting daily prices:

  • < 50 days — shows "Collecting data… N/200 days"
  • 50–199 days — partial signal (50MA only)
  • 200+ days — full Golden/Death Cross logic
Sectors without a tradeable proxy (cash, stables, car) fall back to Portfolio Momentum — the 3-month change in value. This is honest about what it measures: your allocation is shifting, not the market.

Overall market mode

The active mode is decided by majority vote across sectors with valid signals:

  • If the majority of signals are bull → overall BULL
  • If the majority are bear → overall BEAR
  • Otherwise → CRAB (default, safest)

Dynamic Targets

Each category has three target percentages — one for each market mode. The system uses the set matching the current overall mode.

Example default targets:

CategoryBullCrabBear
stocks60%45%30%
stable_yielding25%43%55%
crypto10%5%2%
cash3%5%10%
car2%2%3%

To edit your bull / crab / bear targets, open the Targets tab in /data — the /targets page itself is read-only and exists to help you compare current allocation against the active target.

Data Editor

/data — the single place to add, edit, and delete every kind of record.

Tabs:

  • Accounts — every account (active + deactivated). Add new, deactivate.
  • Assets — every asset, filterable by type. Add new, deactivate.
  • Snapshots — raw snapshot rows for any date. Pick a date, review values, delete bad rows.
  • Positions — current holdings (read-only here; managed via snapshot upserts).
  • Income — income records. Add new, delete.
  • Expenses — expense records, with optional category and labels. Add new, delete.
  • Targets — bull / crab / bear allocation targets per category. Edit any cell.
  • Categories — expense categories with parent-child hierarchy and optional color. Used in the breakdown chart on /expenses.
  • Labels — orthogonal tags applied to expenses (e.g. vacation-2026, tax-deductible). Many-to-many with expenses; a label in use is soft-deleted, an unused one is permanently removed.

Categorizing expenses

An expense can have one category (with hierarchy, e.g. Food / Restaurants) and any number of labels. The two are independent:

  • Category = "what kind of spending is this" — exclusive, used in the donut chart on /expenses.
  • Labels = "what context does this belong to" — non-exclusive, useful for cutting across categories ("everything tagged vacation-2026").

In the /data expenses form, pick a category from the dropdown and type a comma-separated list of label IDs (the available IDs are listed below the form). Via the API, send {"category_id": 4, "label_ids": [1, 2]} on create or update.

Edit vs View — the rule

The app makes a hard separation between editing and visualization:

  • /data — the only place where you can add, edit, or delete records.
  • Every other page — read-only. Charts, tables, signals, and decisions. No edit chrome competing with the data you're trying to interpret.

Each visualization page links back to its corresponding /data tab via a small "edit in /data →" in the page header. This keeps decision-driving pages clean and makes "where do I edit this?" a single answer instead of a memory test.

Working in /data

Each /data tab follows the same shape:

  • + add row — collapsed by default at the top. Click to expand the add form.
  • Table — every record, with a delete link per row.
  • Targets tab — the only tab with inline editing per cell. Type a value, click ✓ to save.

Smart delete

  • Accounts & Assets — try to permanently delete; if the row is referenced by snapshots, positions, or transactions, it falls back to soft-delete (active=0). Historical data is preserved.
  • Income, Expenses, Snapshots, Targets — hard delete (permanently removed).

Pricefeed Microservice

A separate binary that fetches live prices and updates the database.

Setup

  1. Get a free CoinGecko API key from coingecko.com
  2. Add to .env: COINGECKO_API_KEY=your_key_here
  3. Run once: cargo run --bin nworth-feed -- --once
  4. Or run on a schedule: cargo run --bin nworth-feed -- --loop 3600 (every hour)

What it does

  1. Fetches crypto prices from CoinGecko (uses coingecko_id on each asset)
  2. Fetches stock prices from Yahoo Finance (uses yahoo_ticker)
  3. Fetches FX rates from exchangerate.host (EUR, GBP, PYG)
  4. Updates positions.last_price and value_usd
  5. Creates daily snapshot rows from positions × latest prices
